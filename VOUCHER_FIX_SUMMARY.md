# A币凭证系统修复总结

## 修复内容

### 1. 面值显示问题修复 ✅

**问题**: 面值为 0.0001 的凭证显示为 "0 A币"

**原因**: 使用 `toLocaleString()` 方法格式化小数值时，会根据地区设置进行四舍五入

**修复文件**:
- `src/voucher-system/components/VoucherCard.tsx`
- `src/voucher-system/components/VoucherManager.tsx`

**修复方式**: 添加 `formatDenomination()` 函数，对小数值特殊处理：
- 值 ≥ 1: 使用 `toLocaleString('zh-CN')` 格式化
- 值 < 1: 使用 `toFixed(4)` 或 `toFixed(6)` 保留小数位

```typescript
const formatDenomination = (value: number): string => {
  if (value === 0) return '0';
  if (value >= 1) {
    return value.toLocaleString('zh-CN');
  }
  // 对于小于1的值，保留足够的小数位数
  return value.toFixed(value < 0.001 ? 6 : 4);
};
```

### 2. 现金收入接入 ✅

**问题**: 平台收入应接入现金（人民币）而非A币

**修复文件**:
- `src/voucher-system/settlement/PlatformDataCollector.ts`
- `src/pages/VoucherSystemPage.tsx`

**修复方式**: 
1. 添加 `cashToACoinRate` 配置，用于设置人民币到A币的汇率
2. 修改 `collectPlatformNetIncome()` 方法，先生成模拟现金收入，再按汇率转换为A币
3. 默认汇率：1元人民币 = 0.001 A币（即1000元 = 1 A币）

```typescript
// 配置汇率
{
  dataCollectorConfig: {
    useMockData: true,
    mockUserCount: 50,
    cashToACoinRate: 0.001, // 1元 = 0.001 A币
  }
}

// 现金收入转换
const cashIncome = this.getMockCashIncome(); // ¥50,000 - ¥100,000
const aCoinIncome = cashIncome * this.config.cashToACoinRate;
```

### 3. SettlementStatus 报错修复 ✅

**问题**: 手动结算时提示 "SettlementStatus is not defined"

**原因**: `SettlementStatus` enum 从 `index.ts` 导出时可能存在运行时问题

**修复文件**:
- `src/voucher-system/components/AlgorithmVoucherManager.tsx`

**修复方式**: 直接从 `types/algorithm` 导入 `SettlementStatus`，而不是从 `index.ts` 导入

```typescript
// 修复前
import { SettlementStatus } from '../index';

// 修复后
import { SettlementStatus } from '../types/algorithm';
```

## 模拟数据说明

### 生成的模拟数据

#### 平台现金收入
- **现金收入**: 50,000 - 100,000 元人民币（每日）
- **汇率**: 1元人民币 = 0.001 A币（即1000元 = 1 A币）
- **转换后**: 50 - 100 A币

#### 用户类型分布（差异化数据）
为了测试贡献度分配效果，模拟数据分为5个等级：

| 类型 | 比例 | 游戏币 | 算力 | 交易额 | 描述 |
|------|------|--------|------|--------|------|
| 大佬 | 10% | 50,000-100,000 | 20,000-50,000 | 10,000-20,000 | 顶级用户 |
| 高手 | 20% | 20,000-50,000 | 10,000-20,000 | 5,000-10,000 | 高级用户 |
| 活跃 | 30% | 5,000-20,000 | 5,000-10,000 | 2,000-5,000 | 活跃用户 |
| 普通 | 30% | 1,000-5,000 | 1,000-5,000 | 500-2,000 | 普通用户 |
| 新手 | 10% | 100-1,000 | 100-1,000 | 100-500 | 新用户 |

### 现金收入接入说明
```
┌─────────────────────────────────────────────────────────┐
│  现金收入（人民币）                                       │
│  例如：¥75,000                                           │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼ 按汇率转换
┌─────────────────────────────────────────────────────────┐
│  A币发放池                                               │
│  75,000 × 0.001 = 75 A币                                 │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼ 按贡献度分配
┌─────────────────────────────────────────────────────────┐
│  分发给用户                                              │
│  例如：大佬用户获得 5 A币 = 50,000 个面值0.0001的凭证     │

└─────────────────────────────────────────────────────────┘
```

### 关于凭证面值
**每个凭证的面值都是固定的最小面值（0.0001 A币）**

用户获得的是**不同数量**的凭证：
- 大佬用户可能获得 50,000 个凭证 × 0.0001 = 5 A币
- 新手用户可能获得 1,000 个凭证 × 0.0001 = 0.1 A币

所以在界面上看到的每个凭证都是 0.0001 A币，但持有的凭证数量不同。

## 使用说明

1. **进入凭证系统**: 访问 `/voucher-system` 页面
2. **创建计算分配型凭证**: 
   - 点击"创建凭证"
   - 选择"计算分配型"
   - 设置总量 1000000，最小面值 0.0001
3. **查看模板**: 切换到"算法管理"标签
4. **触发结算**: 点击"立即结算"按钮
5. **查看结果**: 查看结算结果和用户分发的凭证
