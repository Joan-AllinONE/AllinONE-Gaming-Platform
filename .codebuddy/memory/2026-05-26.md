### 核心架构
- 三方治理权重：游戏方40% / 玩家35% / 平台25%（可每游戏独立配置）
- 加权投票计算：VIP玩家权重2x，普通玩家0.5x
- 惩罚机制：警告→冻结资产→限流→下架（可配置违规次数阈值）
- 法定人数 + 通过阈值 + 否决权

### 动态权重公式
voteWeight = BASE_WEIGHT × (活跃度×0.35 + 资产×0.25 + 治理×0.25 + 亲密度×0.15)
每因子范围 [0.2~0.3, 1.0]，总乘数 ≈ [0.85, 1.0]

### 自动执行流程
提案通过 + autoExecute=true → dispatchEvent('proposal-auto-execute') → voucherItemService 监听执行 → 按提案类型分发(NEW_ITEM创建模板+铸造/MINT_ITEM铸造/EDIT_ITEM更新模板等)
