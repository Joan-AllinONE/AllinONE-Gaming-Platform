## 修复道具商店"已售罄"误判 Bug

### 问题
道具凭证生成后，在游戏中心道具商店中显示"已售罄"，但实际上平台池中还有100张凭证可售。

### 根因
`GameStore.tsx` 中"已售罄"和"剩余"判断使用了 `totalSupply - mintedCount`，但 `mintedCount ≥ totalSupply` 只表示"铸造已满额"，不表示"库存卖完了"。实际可购买库存应该查询平台池中的可用凭证数量。

### 修改
- `src/pages/GameStore.tsx` — 2 处改动：
  1. 剩余显示：从 `Math.max(0, totalSupply - mintedCount)` 改为 `voucherItemService.getPoolItemVoucherCount(item.id)`
  2. 已售罄判断：从 `mintedCount >= totalSupply` 改为 `getPoolItemVoucherCount(item.id) === 0`
  3. 补充修复：按钮文字（第1081行）也遗漏了同样的判断，一并修正（第1次改漏了按钮内的文字渲染）

## 投票凭证化 — 将投票接入凭证系统

### 设计方案
混合型投票凭证：分发阶段用即时型语义（提案创建时为每位合格选民发行投票凭证），结算阶段用计算型语义（收集所有凭证，加权计算）。不可转让，仅对新提案生效。

### 改动文件（6个）
- **新建** `src/voucher-system/types/vote.ts` — 投票凭证类型定义（VoteVoucherInfo, VoteCycle, VoteSettlementResult, VoteCycleStatus, VoteSettlementStatus）
- **修改** `src/voucher-system/types.ts` — `VoucherSourceType` 新增 `VOTE = 'vote'`
- **新建** `src/voucher-system/services/VoteVoucherService.ts` — 投票凭证核心服务：startVoteCycle() 批量发行投票凭证 / castVote() 消耗凭证投票 / settleVoteCycle() 结算加权结果 / 查询方法
- **修改** `src/voucher-system/services/VoucherService.ts` — createVoucher() 支持 VOTE 来源类型
- **修改** `src/voucher-system/index.ts` — 导出 VoteVoucherService
- **修改** `src/services/gameProposalService.ts` — createProposal() 自动调用 startVoteCycle()；submitProposalVote() 优先用凭证系统投票（旧提案回退）；finalizeProposal() 委托 settleVoteCycle()；新增 syncVoucherSettlementToProposal() 辅助方法
- **修改** `src/components/voucher-system/GameProposalDetailModal.tsx` — 投票状态/记录显示从凭证系统读取；hasVoted 检查走 VoteVoucherService；voteRecords 合并凭证系统数据

### 关键设计
- 投票凭证面额 = 动态投票权重
- 凭证生命周期：CREATE(ACTIVE) → castVote(FROZEN) → settleVoteCycle(EXPIRED)
- 旧提案 voteRecords 保留，新提案同步写入双份（凭证 + voteRecords）
- 结算幂等：已 COMPLETED 的周期不会重复结算

## 投票凭证区分增强 & 结算后保留策略

### 问题一：凭证区分信息不足
**分析结论**：需要在两个层面增强区分信息：
- 凭证元数据：名称格式 `🗳️ [状态] 游戏名 - 提案标题`，描述含截止时间+提案编号+简述
- `VoteVoucherInfo` 结构：新增 `votingEndAt`、`proposalType`、`proposalDescription` 字段

### 问题二：投票后凭证应保留(EXPIRED)还是销毁(DESTROYED)
**最终结论**：保留 EXPIRED，不销毁。理由：
- 数据可审计性：需追溯投票记录用于治理透明度
- 历史参与率：`voteParticipationRate` 依赖历史数据计算未来权重
- 语义正确：EXPIRED="有效期已过但凭证实体仍存在"，DESTROYED="被主动销毁"
- 现实类比：股东大会/DAO治理/政府选举都保留投票记录

### 改动（2个文件）
- `src/voucher-system/types/vote.ts` — `VoteVoucherInfo` 新增 `votingEndAt`、`proposalType`、`proposalDescription` 三个字段
- `src/voucher-system/services/VoteVoucherService.ts` — 6处改动：
  1. 导入 `GameProposalTypeLabel`
  2. `buildVoteMetadata()` 重写：名称含状态前缀+游戏名+提案标题，描述含截止时间+提案编号+简述，tags动态化含游戏slug+提案类型
  3. `startVoteCycle()` 传入 `votingEndAt`、`proposalType`（中文标签）、`proposalDescription`（截取80字）
  4. `castVote()` 投票后刷新凭证名称为 `[已投票]` 状态
  5. `settleVoteCycle()` 结算后刷新名称为 `[已结算]`，描述追加投票结果（赞成/反对/弃权 · 通过/拒绝）
  6. `getUserVoteVouchers()` 新增可选 `status?: VoucherStatus` 参数，支持按状态筛选
