# TestAcoin 算法模板创建指南

按照以下步骤，通过凭证系统的 UI 手动创建 TestAcoin 测试模板。

---

## 操作入口

1. 进入**凭证系统**页面（路由 `/voucher-system`）
2. 切换到 **「算法管理」** 标签页
3. 点击 **「创建模板」** 按钮

## 参数填写参考

### 基本信息

| 字段 | 填写值 | 说明 |
|---|---|---|
| **模板名称** | `TestAcoin` | 测试用模板 |
| **描述** | y` | 可选 |
| **结算周期** | `每日` | 与原 A币一致 |
| **结算时间** | `00:00` | 默认即可 |

### 面值配置

| 字段 | 填写值 | 说明 |
|---|---|---|
| **最小面值** | `0.01` | 继承原 A币最小单位 |

### 贡献度权重

| 字段 | 填写值 | 说明 |
|---|---|---|
| **游戏币权重** | `0.5` | 与原 A币一致（50%） |
| **算力权重** | `0.3` | 与原 A币一致（30%） |
| **交易额权重** | `0.2` | 与原 A币一致（20%） |

### 发放池配置

| 字段 | 填写值 | 说明 |
|---|---|---|
| **发放池计算模式** | 选择 **「固定总量」** | 不依赖平台收入，直接指定发放量 |
| **固定发放总量** | `500` | 每个结算周期最多发放 500 TestAcoin |

> 注：当前为测试阶段，总量 500 在 localStorage 中约占用 12KB，完全无压力。

---

## 创建成功后验证流程

创建模板后，可以按以下步骤验证核心流程：

### 1. 确认模板列表
- 刷新「算法管理」页面
- TestAcoin 模板应出现在模板列表中，状态为「已启用」

### 2. 手动触发结算
- 选中 TestAcoin 模板，点击 **「触发结算」** 按钮（需要管理员权限）
- 系统会自动执行：
  - 创建结算周期
  - 收集用户贡献度数据
  - 计算分配结果
  - 发行算法凭证

### 3. 查看结算结果
- 切换到 **「结算周期」** 标签页，查看结算状态
- 状态应为：待结算 → 收集中 → 计算中 → 发行中 → **已完成**

### 4. 查看用户凭证
- 切换到 **「我的凭证」** 标签页
- 应看到新收到的 `sourceType = algorithm` 的 TestAcoin 凭证
- 点击凭证可查看交易历史和 `AlgorithmVoucherInfo` 详细快照

### 5. 凭证流转测试
- 尝试将 TestAcoin 凭证转入/转出
- 查看交易历史是否可追溯

### 6. 面值转换验证
- 最小面值 0.01，验证用户获得的面值是否为 0.01 的整数倍

---

## 注意事项
- TestAcoin 不会自动触发结算，需要手动触发
- 总量 500 用完即止，可手动重置或重新创建设置更大总量
- 测试数据存储在 localStorage 中，清空浏览器缓存会丢失
