技能文档
1. 什么是技能
技能可以理解为是开发者给Agent写好的「工具包」，让它能做之前做不到的事情。工具包含了做事的方法步骤、需要用到的工具、以及现成的模板资源等。同一个基座Agent接入不同的工具包，就可以实现不同的能力满足各类需求。
可以将专业流程知识封装为可复用的模块化资源，让通用模型快速变身垂直领域专家。它本质是一个包含SKILL.md文件的文件夹，内部可以收纳指令文档、运行脚本和任务相关资源。
•	标准化流程：将SOP文档化，确保任务执行一致性
•	可执行脚本：内置Python/Bash代码，实现确定性计算操作
•	资源附件：可内嵌参考资料、模板文件等静态资源

适用于固化使用流程，或者期望分步执行，再生成技能的场景
核心逻辑是【先执行再打包】：先通过自然对话完成任务，确认各环节执行效果符合预期，然后直接向agent提出需求：【帮我把上述流程生成技能】。

2. 技能的工作机制
技能本质上是一个遵循统一规范的结构化文件夹，用于描述完成某类任务所需的标准作业流程（SOP）、方法论和参考模板。
为保证对话效率，智能体不会一次性加载技能的全部内容，而是采用渐进式披露机制：在对话中判断当前任务是否需要某个技能，仅按需加载与任务相关的说明、规则或代码，从而避免上下文窗口过载。
一个标准技能通常包含以下部分：
代码块
1.	my-skill/
2.	SKILL.md # 【必选】技能说明与元数据
3.	scripts/ # 【可选】可执行代码
4.	references/ # 【可选】参考文献
5.	assets/ # 【可选】模板与静态资源

各目录说明如下：
•	SKILL.md（必选）
技能的入口文件，用于定义技能的名称、用途、适用场景以及调用条件。
智能体会始终读取其中的元数据，以判断是否以及如何使用该技能。
•	scripts/（可选）
存放可执行代码，用于完成确定性的计算或处理任务。
•	references/（可选）
存放参考资料，如第三方 API 文档、规范说明或设计文档，供智能体按需查阅。
•	assets/（可选）
存放模板与静态资源，用于辅助任务执行或直接作为输出的一部分。
